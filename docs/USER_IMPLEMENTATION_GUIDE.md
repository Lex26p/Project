# Dispatcher — краткая инструкция по реализации

1. Открой `./IMPLEMENTATION_STATE.md` и выбери следующий разрешённый sprint.
2. Передай AI-разработчику актуальный commit, sprint ID и `./AI_IMPLEMENTATION_RULES.md`.
3. Примени подготовленные изменения и выполни указанные build/test commands.
4. Ошибки верни модели для исправления в том же sprint scope.
5. После успешной проверки создай commit и используй его как базу следующего sprint.
6. Не переходи через незакрытый gate и не объединяй несколько owner-модулей в одно задание без зависимости из roadmap.

Основной порядок: `./DISPATCHER_IMPLEMENTATION_ROADMAP.md`. Исполняемые задания: `./DISPATCHER_SPRINT_CATALOG.md`.
